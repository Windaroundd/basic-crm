# thimut

Opinionated React starter for vibe coders. Clone it, fill in `.env`, start
shipping features.

## Stack

- **Vite** + **React 19** + **TypeScript** (strict)
- **Tailwind CSS v4** + **shadcn/ui** (new-york, neutral, with dark mode)
- **TanStack Router** (file-based, type-safe routing + code splitting)
- **TanStack Query** (server state + caching)
- **TanStack Table** (headless tables)
- **React Hook Form** + **Zod** (form state + validation)
- **Zustand** (client state, with localStorage persist)
- **Supabase** (Postgres + Auth + Storage + Realtime)
- **Recharts** (charts)
- **Vitest** + **Testing Library** (unit + component tests)
- **Prettier** + **ESLint** + **Husky** + **lint-staged** (DX)

## Quick start

```bash
# 1. Install deps
npm install

# 2. Set up env
cp .env.example .env
# → Fill VITE_SUPABASE_URL and VITE_SUPABASE_ANON_KEY
# → Get them at https://supabase.com/dashboard/project/_/settings/api

# 3. Run dev server
npm run dev
```

## Scripts

| Command                  | What it does                                          |
| ------------------------ | ----------------------------------------------------- |
| `npm run dev`            | Vite dev server with HMR                              |
| `npm run build`          | Generate routes + type-check + production build       |
| `npm run preview`        | Preview production build locally                      |
| `npm test`               | Vitest watch mode                                     |
| `npm run test:run`       | Run tests once (CI)                                   |
| `npm run test:ui`        | Vitest UI dashboard                                   |
| `npm run format`         | Format all files with Prettier                        |
| `npm run format:check`   | Check formatting (CI)                                 |
| `npm run lint`           | Run ESLint                                            |
| `npm run db:start`       | Start local Supabase stack (needs Docker)             |
| `npm run db:reset`       | Reset local DB + reapply migrations                   |
| `npm run db:diff <name>` | Create migration from current schema diff             |
| `npm run db:gen:local`   | Generate TS types from local DB                       |
| `npm run db:gen:remote`  | Generate TS types from linked remote project          |
| `npm run routes:watch`   | Watch + regenerate route tree (usually auto via vite) |

## Folder structure

```
src/
├── components/
│   ├── ui/              ← shadcn primitives (don't edit, add more via `npx shadcn add`)
│   ├── layout/          ← Header, Sidebar, ThemeToggle, ThemeProvider
│   └── common/          ← Spinner, EmptyState, ErrorBoundary, etc.
├── features/            ← 🌟 Main place for feature code (one folder per domain)
│   └── auth/            ← Example feature
│       ├── api.ts         ← Supabase calls (signIn, signOut, getSession)
│       ├── schemas.ts     ← Zod schemas + inferred types
│       ├── components/    ← Feature-specific components (SignInForm)
│       ├── hooks/         ← Feature-specific hooks (useSession)
│       └── index.ts       ← Barrel export — what other code can import
├── hooks/               ← Shared hooks (useDebounce, useMediaQuery)
├── lib/                 ← Third-party clients + cross-cutting utils
│   ├── env.ts             ← Zod-validated env vars
│   ├── supabase.ts        ← Typed Supabase client
│   ├── query-client.ts    ← React Query client config
│   └── utils.ts           ← cn() helper for Tailwind
├── routes/              ← TanStack Router files (THIN — just wire features)
│   ├── __root.tsx         ← Root layout (Header + Outlet + Providers)
│   ├── index.tsx          ← /
│   ├── dashboard.tsx      ← /dashboard
│   └── signin.tsx         ← /signin
├── stores/              ← Zustand global stores (UI state only — NOT server data)
│   └── ui-store.ts
├── types/               ← Shared types
│   └── database.ts        ← Generated by `npm run db:gen:*`
├── test/                ← Vitest setup
│   └── setup.ts
├── main.tsx             ← App entry (providers + router)
├── index.css            ← Tailwind + shadcn theme tokens
└── routeTree.gen.ts     ← Auto-generated by tanstack/router-plugin (don't edit)
```

## How to add things

### Add a new feature

```bash
mkdir -p src/features/posts/{components,hooks}
```

Then:

- `src/features/posts/schemas.ts` — Zod schemas
- `src/features/posts/api.ts` — Supabase queries/mutations
- `src/features/posts/hooks/use-posts.ts` — React Query hook
- `src/features/posts/components/post-list.tsx` — UI component
- `src/features/posts/index.ts` — barrel export

### Add a new route

Create a file in `src/routes/`. TanStack Router auto-detects it:

- `src/routes/posts.tsx` → `/posts`
- `src/routes/posts/$id.tsx` → `/posts/:id` (dynamic)
- `src/routes/posts/index.tsx` + `src/routes/posts/$id.tsx` → nested

Keep route files **thin** — just import feature components:

```tsx
import { createFileRoute } from '@tanstack/react-router'
import { PostList } from '@/features/posts'

export const Route = createFileRoute('/posts')({ component: PostList })
```

### Add a new shadcn component

```bash
npx shadcn@latest add dialog calendar command
```

They land in `src/components/ui/`. Don't edit them by hand — re-run the
command to update.

### Add a new Zustand store

For **global UI state only** (sidebar, modals shared across routes, theme
overrides). Don't put server data here — that's React Query's job.

```ts
// src/stores/cart-store.ts
import { create } from 'zustand'
export const useCartStore = create((set) => ({ items: [], add: ... }))
```

## Rules of thumb

- **Features don't import from features.** If two features need the same
  thing, lift it to `lib/`, `hooks/`, or `components/common/`.
- **Routes are thin.** A route file should mostly just import a component
  from a feature and render it.
- **Server state → React Query.** Don't cache API responses in Zustand or
  useState — let React Query handle invalidation, refetching, etc.
- **Client state → Zustand or useState.** `useState` if scoped to a
  component. Zustand if shared across distant components.
- **Form state → React Hook Form.** With Zod resolver for validation.
- **URL state → TanStack Router search params.** Filters, pagination, tabs
  often belong here so URLs are shareable.

## Supabase setup

### Cloud (recommended for first project)

1. Create project at https://supabase.com/dashboard
2. Settings → API → copy URL + anon key into `.env`
3. (Optional) `npx supabase login && npx supabase link --project-ref <ref>`
   so you can run `npm run db:gen:remote`

### Local (needs Docker)

1. Install Docker Desktop
2. `npm run db:start` — starts local Postgres + Auth + Storage + Studio
3. Copy the printed URL + anon key into `.env`
4. Open Studio at http://localhost:54323

### Generate types after schema changes

```bash
npm run db:gen:local    # from local stack
npm run db:gen:remote   # from linked remote project
```
